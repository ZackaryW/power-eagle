# Plugin development and testing use ~/.powereagle/extensions/{pluginId}/ with plugin.json plus main.js or main.py and console or stdout/stderr debugging.

## Context
- The docs define a deliberately simple local plugin layout that mirrors the production installation format.
- Developer workflow is based on direct file placement and runtime inspection rather than a custom plugin build pipeline.

## Content
- Plugins are developed in the Power Eagle extensions directory using a plugin.json manifest plus a main.js or main.py entrypoint.
- JavaScript plugin debugging primarily uses the browser or extension console.
- Python plugin debugging uses streamed stdout and stderr plus the filtered callback behavior handled by the runner and evaler.
- Manual testing consists of loading the plugin in Power Eagle and exercising its UI or event-driven behavior.

## Consequences / notes
- Documentation and tooling should preserve the direct-file development workflow.
- Debugging guidance should stay split between JavaScript console inspection and Python process output.
