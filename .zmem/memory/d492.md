# Zip extraction must use native system tooling such as PowerShell or unzip instead of bundled zip libraries.

## Context
- The project intentionally avoids bringing in an external zip library for archive extraction.
- The legacy notes frame this as a reliability and dependency-management choice.

## Content
- Windows extraction should use PowerShell and macOS extraction should use unzip or equivalent native tooling.
- Archive handling is expected to rely on system APIs that are already present in the target environment.
- This keeps the dependency footprint small and avoids a second zip abstraction inside the project.

## Consequences / notes
- Changes to the installation pipeline should preserve the system-tooling approach unless a stronger technical constraint emerges.
- Cross-platform archive behavior should be tested through the native command paths the product actually uses.
