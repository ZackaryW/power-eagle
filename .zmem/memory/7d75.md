# The app is a React-based extension manager coordinating plugin discovery, loading, execution, management, downloads, and Python evaluation.

## Context
- The system is organized around focused manager modules instead of a monolithic plugin runtime.
- Legacy architecture notes describe a single application flow that starts in the UI and fans out into specialized managers.

## Content
- App.tsx drives an extension manager that delegates to discovery, loader, executor, management, download, and Python execution components.
- Installed plugins live under the Power Eagle extensions directory and are discovered from disk before being loaded and executed.
- JavaScript plugins run through an isolated execution path with injected Eagle and Power SDK context.
- Python plugins run through a dedicated runner and callback-aware evaluator that bridge Python code back into Eagle APIs.
- Download and installation are separate concerns from execution so archive validation and filesystem operations remain contained.

## Consequences / notes
- Cross-cutting changes should respect the current module boundaries unless there is a strong reason to merge responsibilities.
- Queries about system behavior should usually start with the manager layer responsible for that phase of the plugin lifecycle.
