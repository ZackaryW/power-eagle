# Preserve per-plugin isolation through dedicated DOM containers, prefixed storage, and cleanup of listeners and resources.

## Context
- Multiple plugins can coexist inside the same Eagle environment, so collisions in DOM, storage, and event handling are a real risk.
- The legacy memory-bank describes isolation as a core safety and maintainability principle rather than an optional implementation detail.

## Content
- Each plugin receives its own DOM container, plugin-scoped storage keys, and tracked resources for cleanup.
- Execution contexts are fresh per run so plugin state and listeners do not leak across switches.
- Cleanup is part of the contract for safe plugin management and removal.

## Consequences / notes
- New SDK features should preserve plugin-level isolation boundaries.
- Any shortcut that bypasses prefixed storage or shared DOM ownership introduces conflict risk and should be treated cautiously.
