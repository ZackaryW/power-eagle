# SDK terminology now explicitly separates the host-provided eagle runtime from the HTTP webapi client, with exported types that preserve the distinction for plugin authors and internal code.

## Context
- The project had been using Eagle API language loosely enough that host-provided runtime methods and the SDK webapi HTTP client could be read as the same surface.
- Recent cleanup renamed SDK types and comments so the distinction is explicit in code and exports.

## Content
- HostEagleAPI now represents the runtime object injected by the Eagle host environment.
- EagleAPI remains as a backward-compatible alias rather than the primary conceptual name.
- webapi remains the separate HTTP client surface, and WebEagleApi is exported as its own SDK type.
- PluginManager now passes the injected host runtime through explicitly instead of implying that the web client and runtime object are interchangeable.

## Consequences / notes
- Future SDK changes should keep host runtime behavior and web API behavior as separate identities in both naming and docs.
- Plugin author guidance should explain when to use the host eagle object versus powersdk.webapi.
