# The Python callback system provides secure bidirectional Python-to-Eagle communication by filtering stderr signals and validating API tokens.

## Context
- The legacy active-context and progress notes treat the callback system as the biggest recent capability jump in the project.
- Before this work, Python support existed, but direct Eagle API callbacks from Python were much weaker.

## Content
- Python scripts emit callback signals over stderr using a structured token, plugin ID, and method payload.
- PythonScriptEvaler intercepts those signals in real time, removes them from user-visible output, validates the current Eagle token, and executes the corresponding Eagle API call in a safe context.
- The callback path is intentionally limited to fire-and-forget operations, while methods needing return values are excluded from this mechanism.
- The result is clean Python output plus practical bidirectional integration between Python automation and Eagle operations.

## Consequences / notes
- Python support should be understood as a full callback-capable integration layer, not just child-process script execution.
- Security validation and output filtering are part of the feature, not incidental implementation details.
- Any attempt to expand the protocol into a request-response channel would need a different design from the current stderr signaling model.
