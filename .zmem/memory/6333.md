# Deliver a userscript-style plugin system for Eagle with instant URL-based installation and updates.

## Context
- Eagle has a capable native extension model, but it is too heavy for quick utilities and one-off automation.
- The project exists to give Eagle users a Tampermonkey-like experience where a shared URL becomes a working plugin within seconds.

## Content
- The core product promise is instant installation, easy updates, and low-friction sharing of lightweight plugins.
- The plugin contract stays deliberately small: a manifest plus a single JavaScript or Python entrypoint.
- The platform prioritizes community distribution of small utilities rather than full application-style extensions.

## Consequences / notes
- Product and architecture decisions should favor install speed, authoring simplicity, and URL-based distribution.
- Features that add extension-style ceremony without improving the userscript workflow work against the core goal.
