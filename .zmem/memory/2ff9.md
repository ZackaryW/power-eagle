# PluginDownload validates and installs zip archives, then PluginDiscovery rescans installed extensions for activation.

## Context
- Installation is URL-driven, so the download path must safely bridge remote archives into the local extensions directory.
- The legacy flow separates archive handling from discovery and execution.

## Content
- PluginDownload is responsible for fetching archives, validating expected contents, extracting them, and moving them into the installation directory.
- After installation, PluginDiscovery rescans the extensions location so the application can surface the new or updated plugin.
- Re-installing the same plugin ID is part of the intended update flow and should remain straightforward.

## Consequences / notes
- Validation should stay close to the download and extraction path.
- Discovery should remain the authoritative source of installed plugin state after filesystem changes.
