# The stderr callback protocol is fire-and-forget; methods that require return values are excluded from callback execution.

## Context
- The callback protocol is designed for side-effecting Eagle actions emitted from Python over stderr.
- The evaler documentation explicitly excludes methods that depend on returning data to the Python caller.

## Content
- Callback lines use the $$$token$$$plugin$$$method(args) stderr format and are consumed by the evaluator instead of surfacing in final stderr output.
- The protocol is suitable for operations such as creating folders or showing notifications where a side effect matters more than a return payload.
- Methods that require return values are intentionally excluded from this channel and must use another access path.
- Filtering callback lines out of stderr is part of the protocol contract, not just a logging convenience.

## Consequences / notes
- Extending callback support should respect the fire-and-forget nature of the current protocol.
- Return-value workflows should not be bolted onto the stderr channel without redesigning the contract.
