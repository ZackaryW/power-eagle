# Development uses npm or pnpm with a Vite TypeScript build, while runtime filesystem and process features come from Eagle's Electron environment.

## Context
- Developers build the project with standard JavaScript tooling, but plugin runtime behavior depends on Eagle hosting the extension.
- The technical notes capture both the local build loop and the host-provided runtime capabilities.

## Content
- The repository is built with npm or pnpm and a Vite plus TypeScript toolchain.
- Linting and typing are part of the normal development environment.
- Filesystem access, child-process execution, and Eagle API integration are available at runtime through the Eagle Electron environment.
- Plugin authors test against the installed-style directory layout under ~/.powereagle/extensions/{pluginId}/ using plugin.json plus main.js or main.py.
- JavaScript debugging centers on console inspection, while Python debugging relies on stdout and stderr streamed through the Python runner.

## Consequences / notes
- Local development instructions should distinguish clearly between repository build steps and host-runtime behavior.
- Bugs that reproduce only inside Eagle should be evaluated separately from plain frontend build issues.
- Plugin author setup guidance should mirror the runtime directory structure to keep onboarding simple.
