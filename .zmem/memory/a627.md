# Python script plugins run as separate processes and receive Eagle state through the POWEREAGLE_CONTEXT JSON environment variable.

## Context
- Python plugins do not share the in-process JavaScript plugin runtime, so they need a transport for Eagle state.
- The documentation defines POWEREAGLE_CONTEXT as the canonical bridge for folders, items, library info, and the web token.

## Content
- Python scripts are launched as separate processes rather than receiving the JavaScript plugin context object directly.
- The runner serializes Eagle state into the POWEREAGLE_CONTEXT environment variable as JSON.
- That JSON includes selected folders, selected items, library information, and the web API token.
- This design lets Python plugins use the wider Python ecosystem while still operating on current Eagle state.

## Consequences / notes
- Changes to Python context shape should be treated as a compatibility concern for plugin authors.
- Python examples and tooling should assume environment-driven context hydration rather than in-process SDK access.
